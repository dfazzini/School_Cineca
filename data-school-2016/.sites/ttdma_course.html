Setup Instructions

Read carefully this PDF file to reach the minimum environment for our course.

Launch the clusters

If you already setup your system, you just go into the GIT project directory (e.g. $HOME/lectures) and manage to launchwith docker-compose

  • MapReduce on Hadoop with Mrjob:
    $ docker-compose up hadoopclient
  • Spark Context with a Master and a Worker:
    $ docker-compose up sparkclient

Access the notebook web pages

To interact with Hadoop or Spark you have to use or write a Jupyter Notebook. Go to http://localhost (or your server IP) and start your work.

Stop and clean resources

To stop running containers you may press CTRL+c from the terminal where you launched docker-compose.

If experiencing problems, to remove all containers on your host you can use the docker command:
$ docker rm -f -v $(docker ps -a -q)

Note: all data will maintain changes as it is provided via docker volume mount, thus will not be deleted with the remove command.
